1. Architecture Overview
This document presents the finalized information architecture for the Incorporate123 website redesign. The architecture consolidates approximately 325 unique current URLs into a focused ~75-page structure organized around four intent-based content pillars: Business Privacy, Asset Protection, Company Formation, and Compliance.
The architecture has been validated against 392 crawled URLs mapped to destinations, 5 competitors analyzed for structural gaps, 5 user journeys confirmed through flow analysis, and SEO keyword data identifying content priorities. Every page, URL, and structural decision is traceable to audit findings, competitive gaps, or user intent data.

3. URL Strategy
All URLs use a flat structure with no deep directory hierarchy. The pillar/cluster hierarchy is established entirely through internal linking, not URL paths. This keeps URLs clean, shareable, and future-proof  --  pages can be reorganized between pillars without breaking URLs.

 Pattern: /descriptive-slug/ (e.g., /wyoming-private-incorporation/ not /services/wyoming/private-incorporation/)
 Exception: Offshore pages retain /offshore/ prefix for content grouping (e.g., /offshore/nevis/)
 Exception: Checkout flow uses /checkout/ prefix (e.g., /checkout/configure/)
 Trailing slashes: Enforced consistently with server-level normalization
 Canonicals: Self-referencing canonical tag on every page to prevent duplicate content issues
