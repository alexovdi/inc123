## The conversion funnel

**Hook → Trust → Agitate → Educate → Guided Solution → Process → Transparent Price → Why Us → Social Proof → Close**

Each section earns the right to the next. Swapping any of them makes the following section feel premature:

- **Hook before agitate** — can't agitate a problem no one's engaged with
- **Agitate before educate** — otherwise education lands as trivia, not urgency
- **Educate before price** — otherwise the price is meaningless
- **Price before justification** — otherwise the "why us" feels defensive
- **Justification before proof** — otherwise claims feel unsupported
- **Proof before close** — otherwise the close lands on a skeptical reader

---

## Two parallel conversion paths

The homepage intentionally supports two distinct user types without forcing them down the same path:

- **Guided path (lowest friction):** Hero CTA → Wizard → Result → Checkout
- **Transparent path (researchers):** Hero → Pillar Cards → Package Comparison → Tier page → Checkout

Both lead to the same checkout; the user picks based on whether they want to be told or want to decide.

---

## Key ordering principles encoded in this layout

1. **Hook before everything.** Dark hero sets brand tone and makes one clear promise in 3 seconds.
2. **Trust before pain.** A visible trust ticker between hero and problem section buys permission to agitate.
3. **Problem before solution.** Privacy Bridge earns the right to sell by establishing urgency.
4. **Browse before quiz.** Pillar cards give researchers an exit before the wizard forces a decision.
5. **Guided before transparent.** Wizard is first conversion instrument; comparison grid is the fallback.
6. **Process before price.** "How It Works" reduces effort anxiety before the pricing grid appears.
7. **Price before justification.** Differentiators answer the "why does it cost this much" question _after_ the number is on screen.
8. **Claims before proof.** Testimonials validate the differentiators in the user's own voice.
9. **Dark bookend.** Hero opens dark, Final CTA closes dark — rhythmic visual framing of the experience.
10. **Persistent CTA on mobile.** Sticky bar gives mobile users a one-tap entry from any scroll position.
