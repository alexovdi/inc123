## Wireframe Improvement Notes

### 1. Reading Time Estimate May Create Friction for Informed Buyers

"8 min read" signals "this is a long page" — which is great for researchers but may push Informed Buyers (Journey D) away from the content entirely. Informed Buyers want to skip education and reach packages in 1–2 clicks.

**Recommendation:** Keep the reading time badge but ensure the hero's secondary CTA ("Compare Wyoming vs Nevada →") and the sidebar Package Shortcut are prominent enough that Informed Buyers can bypass the content entirely.

### 2. Six Content Sections Before the Package CTA Is Deep

The wireframe has six H2 content sections before the Package CTA Block. For a researcher this is ideal, but conversion-focused visitors may lose patience. The current wireframe places an inline Package Shortcut from the sidebar after Section 5 on mobile — this is correct.

**Recommendation:** On desktop, ensure the sidebar Package Shortcut card is visually prominent and uses amber CTA styling. On mobile, consider placing the inline package shortcut after Section 3 (the comparison table) rather than Section 5, since the comparison table is a natural decision point.

### 3. Missing "Key Takeaway" Summary Box

Long-form cluster pages benefit from a TL;DR summary box near the top — a highlighted card with 3–4 bullet points summarizing the page's core message. This serves scanners who want the essence without reading 8 minutes of content, and anchors researchers before they dive in.

**Recommendation:** Add a "Key Takeaways" box between the hero and Section 1, containing 3–4 bullet-style takeaways. Example for Anonymous LLC: "Your name never appears on public record · Year-round nominees maintain privacy on every filing · Wyoming is the lowest-cost privacy state · Gold packages include everything from $1,275."

### 4. Comparison Table Needs Mobile-First Design Decision

The wireframe offers two mobile options (stacked cards vs. horizontal scroll) but doesn't commit. This decision affects development and should be locked during the wireframe phase, not punted to dev.

**Recommendation:** Commit to stacked factor cards for mobile. Horizontal scroll tables are difficult to discover on mobile (users don't always realize they can swipe), and the comparison table has relatively few columns (3), making stacked cards clean and readable.

## Long-Form Content Sections

Each cluster page has 4–6 H2 content sections following the 80/20 model. Each section has:

- **Visible scanner content:** 3–5 sentences covering the core point
- **Expandable reader content:** 200–600 words going deeper
- **Key Point Callout (optional):** A highlighted box emphasizing a critical fact or differentiator
- **Inline CTA (optional):** Soft link to a related cluster or comparison page

### Section Pattern

| Section Type | Purpose | Example (Anonymous LLC) |
|-------------|---------|------------------------|
| **Definition** | "What is [Topic]?" — explains the concept from zero | "What is an Anonymous LLC?" |
| **Mechanism** | "How [Topic] Works" — step-by-step or process explanation | "How Anonymous LLCs Work" (4 steps) |
| **Comparison** | State or option comparison with table | "Wyoming vs Nevada for Anonymous LLCs" (comparison table) |
| **Audience** | "Who Needs [Topic]?" — segments with icons | "Who Needs an Anonymous LLC?" (4 audience blocks) |
| **Differentiator** | Key competitive distinction | "Year-Round Nominees vs. 'Nominee for a Day'" |
| **Pricing** | Transparent cost breakdown | "How Much Does an Anonymous LLC Cost?" |

### Content Rules for All Cluster Pages

1. **Every H2 section must be independently useful.** A visitor who only reads one section should get value from it alone.
2. **Specific numbers always.** "$1,275 all-inclusive" not "competitive pricing." "$400–$5,000/year" not "competitors charge more."
3. **Comparison tables use real data.** Filing fees, renewal costs, and statutory references must be verified by David.
4. **Expandable sections contain the depth, not the keywords.** Primary keywords should appear in the visible scanner content so search engines index them regardless of progressive disclosure implementation.
5. **One differentiator callout per page minimum.** A highlighted box with a competitive price comparison or feature comparison. These are the "aha moment" elements.

### Voice Notes for All Cluster Content

- Written as an authoritative guide, not a sales page. The tone is educational — a knowledgeable advisor explaining how things work.
- Never assume the reader knows legal terminology. Define "nominee," "charging order," "articles of organization," etc. in context.
- Never use fear language. "Protect your privacy" is fine. "Before someone steals your identity" is not.
- Always cite specific numbers. This builds trust and differentiates from competitors who use vague claims.
- End each content section with a natural transition to the next section or a soft CTA to a related page.

## Package CTA Block

Appears after all content sections, before FAQ. This is the conversion moment — the reader has been educated and is ready to act.

| Element | Content Pattern |
|---------|----------------|
| **Headline** | "Ready to [Action Related to Topic]?" |
| **Subtext** | "All-inclusive [Tier] packages with [key differentiators]." |
| **Two cards** | Relevant packages side by side with price, key includes, and CTAs |
| **Below cards** | "Compare All Packages →" + "Schedule a Consultation →" |

**Anonymous LLC example:** "Ready to Form Your Anonymous LLC?" with Wyoming Gold ($1,275) and Nevada Gold ($1,800).

**Formation cluster example:** "Ready to Start Your Wyoming LLC?" with Wyoming Silver ($875) and Wyoming Gold ($1,275) — showing the upgrade path.

## FAQ Accordion

10–12 questions per cluster page. FAQ schema markup on every question/answer pair.

### Content Rules for FAQ Answers

- 2–5 sentences per answer. Not paragraphs — scannable answers.
- Include a "Learn more →" link to the relevant cluster or pillar page when the question warrants a deeper answer.
- Always include a pricing question ("How much does [topic] cost?") with exact figures.
- Always include a comparison question ("Which state is better for [topic]?") linking to the comparison page.
- Answers should be self-contained — a reader who only sees the FAQ (via Google rich result) should get a complete answer.
