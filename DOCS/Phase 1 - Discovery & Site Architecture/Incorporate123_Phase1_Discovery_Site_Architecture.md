## 1. Executive Summary

This document provides a detailed breakdown of Phase 1: Discovery & Strategy for the Incorporate123 website redesign project. Phase 1 is the foundational workstream upon which all subsequent work depends. It transforms the current geography-first website structure into a validated, intent-first site architecture organized around four content pillars.

> **Phase 1 at a Glance**
> 36 hours • Weeks 1–2 • 7 core tasks • 5 key deliverables • Milestone gate before UX & Design begins

Phase 1 sets the strategic direction for the entire project. No design, development, or SEO work begins until the site architecture is confirmed and approved. The deliverables produced in this phase serve as the binding reference for all downstream workstreams.

## 2. Phase Objectives

The Discovery & Strategy phase exists to answer a single critical question: **What are we building, and why?** Every task in this phase contributes to a validated, approved site architecture that the design, SEO, and development teams can execute against with confidence.

### 2.1 Strategic Objectives

- **Reposition Incorporate123** *from* "Nevada incorporation company" *to* "Privacy, Asset Protection & Business Formation Specialists."
- Validate the four-pillar content architecture (Business Privacy, Asset Protection, Company Formation, Compliance) against actual site data and competitive landscape.
- Map the complete conversion flow: Education → Specifics → Comparison → Package → Conversion.
- Identify SEO authority opportunities in privacy and asset protection keywords where competitors have limited presence.
- Ensure zero content loss during the consolidation from ~171 URLs to ~71 pages.

### 2.2 Tactical Objectives

- Crawl and inventory all ~171 existing URLs, categorizing content to preserve, consolidate, or retire.
- Map ~129 unique content pages to the new 71-page architecture with specific destinations.
- Create a preliminary 301 redirect plan for ~170 URLs to preserve domain authority.
- Define primary user journeys for privacy, asset protection, and formation intents.
- Produce a discovery report with strategic recommendations for client approval.

## 3. Current State Analysis

The current Incorporate123 website contains approximately 171 URLs across 15 template types. A significant portion of these represent duplicate or fragmented content, particularly within the Learning Center FAQ section. Understanding this baseline is essential to the consolidation strategy.

### 3.1 Current Site Metrics

| Metric | Count | Notes |
|---|---|---|
| **Total Current URLs** | ~171 | Including all paths and duplicates |
| **Unique Content Pages** | ~129 | After removing ~42 duplicate FAQ URLs |
| **Current Template Types** | 15 | Many serve only 1–3 pages |
| **Learning Center FAQ Items** | ~111 | Across overlapping category paths |
| **New Site Pages (Target)** | ~71 | Across 8–10 custom templates |
| **301 Redirects Required** | ~170 | Every current URL mapped to new structure |

### 3.2 Why 129 Pages Becomes 71

No content is being lost. The current site's ~129 unique pages include significant duplication and fragmentation. The Learning Center alone has ~111 FAQ items, many appearing under multiple overlapping category URLs. Thin pages that serve only a handful of visitors sit alongside redundant service descriptions covering the same ground from different angles.

The new 71-page architecture consolidates all of this into focused, authoritative pages. FAQ content moves into contextual sections on pillar, cluster, and package pages rather than living as standalone URLs. Overlapping service pages merge into single pages that carry more weight with search engines. Every piece of content worth preserving is mapped to a specific destination, and ~170 redirects ensure no traffic or search authority is lost.

### 3.3 The Core Problem

| Current State | Proposed State |
|---|---|
| Geography-first navigation | Intent-first navigation |
| Packages buried in services menu | Packages as clear conversion endpoints |
| No comparison pages or organized flow | Clear pillar/cluster funnel with comparison pages |
| 1999 design undermines trust | Modern, trust-building design |
| Flat, chaotic site structure | SEO-optimized pillar/cluster architecture |
| Package pages overwhelm with content | Progressive disclosure — scanners & readers both served |

### 4.1 Current Site Audit

**Estimated Effort:** ~6 hours

A comprehensive crawl and inventory of the entire existing website. This audit establishes the baseline from which all consolidation, migration, and redirect decisions are made.

**Activities:**

- Full site crawl of all ~171 URLs using automated tools to capture page structure, metadata, content length, and link relationships.
- Categorize each URL as: preserve (migrate as-is), consolidate (merge with related content), enhance (expand into fuller page), or retire (redirect only, no standalone page needed).
- Identify the 15 existing template types and document which pages use each.
- Catalog all ~111 Learning Center FAQ items, noting duplicates appearing under multiple category paths.
- Document existing internal linking patterns and navigation structure.
- Capture current SEO metadata (title tags, meta descriptions, H1 tags) for all indexable pages.

**Output:**

- Complete URL inventory spreadsheet with status classification for each page.
- Template usage map showing current template-to-page relationships.
- FAQ duplication report identifying overlapping content.

---

### 4.2 Content Mapping

**Estimated Effort:** ~6 hours

The most consequential task in Phase 1. Content mapping translates the strategic pillar/cluster architecture into a concrete, page-by-page plan showing exactly where every piece of existing content lands in the new structure.

**Activities:**

- Map each of the ~129 unique content pages to a specific destination in the 71-page architecture.
- Identify content gaps: new pages required that have no existing content equivalent.
- Define the consolidation plan for fragmented content (e.g., FAQ items rolling into pillar and cluster pages).
- Validate the four-pillar structure against actual content inventory to confirm no orphaned topics.
- Document content that crosses pillars (e.g., Wyoming privacy content relevant to both Privacy and Formation pillars).
- Create the definitive content migration map that development will follow.

**Output:**

- Content migration map: Old URL → New URL with content disposition for every page.
- Gap analysis identifying 23+ new pages to be created.
- Cross-pillar content reference plan.

### 4.4 User Journey Mapping

**Estimated Effort:** ~5 hours

Defines the primary paths visitors will take through the new site architecture, from initial search intent through to package conversion. These journeys inform navigation design, internal linking, and CTA placement.

**Activities:**

- Define the primary conversion flow: Pillar (education) → Cluster (specifics) → Comparison (decision) → Package (conversion).
- Map the privacy-focused journey: Visitor searches for anonymous LLC → Privacy pillar → Anonymous LLC cluster → Best State for Privacy comparison → Wyoming Gold package.
- Map the asset protection journey: Visitor searches for LLC protection → Asset Protection pillar → State-specific cluster → Comparison → Relevant package.
- Map the direct formation journey: Visitor knows what they want → Formation pillar or direct to package.
- Identify cross-pillar navigation needs (e.g., privacy visitor who also needs asset protection).
- Define secondary journeys: returning clients seeking compliance services, offshore inquiries, shelf company shoppers.

**Output:**

- User journey flow diagrams for each primary intent.
- Navigation requirements document for UX phase.
- CTA placement recommendations mapped to journey stages.

- Create a verification checklist for post-launch redirect testing.

**Output:**

- 301 redirect map: Old URL → New URL for all ~170 current URLs.

- Post-launch redirect verification plan.

## 6. URL Strategy & Internal Linking

### 6.1 Flat URL Structure

All URLs use a flat structure with no deep hierarchy. For example: `/wyoming-private-incorporation/` instead of `/services/wyoming-private-incorporation/`. The pillar/cluster hierarchy is established through internal linking, not folder structure. This keeps URLs clean, shareable, and future-proof.

### 6.2 Internal Linking Map

| From | Links To |
|---|---|
| **Pillar pages** | All related clusters, comparison pages, relevant packages |
| **Cluster pages** | Back to pillar, related clusters, relevant comparison, relevant package |
| **Comparison pages** | Back to pillar, to relevant packages |
| **Package pages** | Related packages ("Also consider..."), back to pillar |
| **State hubs** | All content for that state across all pillars |

## 8. Milestone Gate & Approval

Phase 1 concludes with a formal milestone checkpoint. This is the gate that unlocks all subsequent workstreams. No UX prototyping, visual design, SEO specification, or development work begins until this milestone is achieved.

> **Milestone 1: Discovery Complete**
> **Deliverables:** Finalized site architecture, content migration map, redirect plan draft, discovery report, positioning framework.
> **Approval Required:** Client sign-off on site architecture.
> **Unlocks:** UX & Interactive Prototyping + Visual Design (Weeks 2–5).

### 8.1 Client Responsibilities During Phase 1

- Provide access to the current website, hosting environment, and domain registrar.
- Provide viewer/editor access to existing Google Analytics 4 and Google Search Console properties.
- Provide any existing brand assets (logos, color preferences, fonts) and content to be preserved.
- Designate a single point of contact with authority to approve deliverables.
- Provide timely feedback on deliverables within 3–5 business days of submission.
- Make decisions on open strategic questions (package structure, domain consolidation, compliance pillar scope).

### 8.2 What Happens After Approval

Once the site architecture is confirmed, the project transitions into parallel workstreams. UX prototyping and visual design begin immediately (Weeks 2–5), with SEO specification work starting as soon as the structure is locked (Week 3). The approved Phase 1 deliverables serve as the authoritative reference for all downstream decisions.

|---|---|---|
| **UX & Interactive Prototyping** | 2–5 | Finalized site architecture, user journey maps |
| **Visual Design & Design System** | 2–5 | Approved prototypes (rolling), positioning framework |
